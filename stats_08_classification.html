分类方法

决策树（熵/基尼/剪枝）、随机森林（Bagging/OOB）、XGBoost / LightGBM、SVM（最大间隔/核技巧）、KNN、朴素贝叶斯、SMOTE 类别不平衡。回答「这个样本属于哪一类？」

竞赛真题场景

Q：国赛 B 题常考：根据多个特征判断样本类别（如根据化学成分判断葡萄酒产地、根据消费行为判断用户类型）。
Q：美赛 MCM C 题：对亚马逊评论进行情感分类（正面/负面），一等奖论文使用随机森林 + TF-IDF 特征。
Q：泰坦尼克号生存预测（Kaggle 经典）：根据乘客属性（舱位、性别、年龄）预测是否生存。
→ 竞赛中，分类问题的核心不只是准确率，而是模型的可解释性——评委需要知道「为什么这样分类」。

8.1 决策树

核心思想

决策树通过递归二分将特征空间划分为矩形区域。每次分裂贪心地选择当前最优特征和切分点（局部最优而非全局最优），使得子节点尽可能「纯」——即同一类样本聚集在一起。最终生成一棵可以翻译为 if-then 规则的树。

8.1.1 信息增益与熵

信息熵衡量一个节点的不纯度。设节点 \(t\) 中第 \(k\) 类样本的比例为 \(\hat{p}_{k}\)，则：

\[ H(t) = -\sum_{k=1}^{K} \hat{p}_{k} \log_2 \hat{p}_{k} \]

当所有类别等概率时，熵最大（\(H = \log_2 K\)）；当所有样本属于同一类时，熵最小（\(H = 0\)）。

推导：为什么信息论中常用 \(\log_2\)？

对数底数决定信息量的单位：\(\log_2\) → bit，\(\ln\) → nat，\(\log_{10}\) → dit。不同底数只会对熵和信息增益乘上一个正常数，不会改变候选分裂的优劣排序——因此选择哪个底数不影响决策树的生长结果。CART 虽是二叉分裂，但使用 \(\log_2\) 并非因为「分裂是二分问题」，而只是一种约定（bit 是信息论的标准单位）。信息增益定义为：

\[ \text{Gain}(t, s) = H(t) - \left( \frac{n_L}{n} H(t_L) + \frac{n_R}{n} H(t_R) \right) \]

其中 \(s\) 是分裂点，\(t_L, t_R\) 是左右子节点，\(n_L, n_R\) 是子节点样本数。我们选择使 Gain 最大化的特征和分裂点。

8.1.2 基尼不纯度

CART（Classification and Regression Tree）使用基尼系数代替熵：

\[ G(t) = 1 - \sum_{k=1}^{K} \hat{p}_{k}^2 = \sum_{k \neq j} \hat{p}_{k} \hat{p}_{j} \]

基尼系数可以理解为：从节点 \(t\) 中随机抽取两个样本，它们属于不同类别的概率。熵和基尼在大多数情况下给出相似的分裂结果，但基尼计算更快（不需要对数运算）。

竞赛提示：CART 默认使用基尼系数，且每个节点只产生二叉树（binary split）。这避免了 ID3/C4.5 的多路分裂导致的数据碎片化问题。在数模竞赛中，推荐使用 CART（sklearn 的默认决策树实现）。

8.1.3 剪枝策略

完全生长的决策树几乎必然过拟合（每个叶子只有一个样本，训练误差为 0）。剪枝是控制复杂度的关键：

代价复杂度剪枝 (Cost-Complexity Pruning)

定义代价函数：

\[ R_{\alpha}(T) = R(T) + \alpha |T| \]

其中 \(R(T)\) 是训练误差（如误分类率），\(|T|\) 是叶子节点数，\(\alpha\) 是复杂度惩罚参数。\(\alpha = 0\) 给出完全生长的树，\(\alpha \to \infty\) 给出只有根节点的树。通过交叉验证选择最优 \(\alpha\)。

常见陷阱：决策树对训练数据的微小变化极度敏感——增加或删除一个样本可能导致完全不同的树结构。这被称为高方差问题。随机森林正是为了解决这个问题而设计的。

8.2 随机森林

核心思想

随机森林 = Bagging + 随机特征选择。训练多棵决策树，每棵树在随机抽样的数据子集上训练，且每次分裂只考虑随机选取的特征子集。最终通过投票（分类）或平均（回归）来预测。

8.2.1 Bootstrap Aggregating (Bagging)

从原始训练集 \(D = \{(x_1,y_1), \ldots, (x_n,y_n)\}\) 中，有放回地抽取 \(n\) 个样本，形成 Bootstrap 样本 \(D^*\)。重复 \(B\) 次，训练 \(B\) 棵树。最终预测：

\[ \hat{f}_{\text{rf}}(x) = \frac{1}{B} \sum_{b=1}^{B} \hat{f}_b(x) \quad \text{(回归)} \qquad \hat{f}_{\text{rf}}(x) = \text{majority vote}\{\hat{f}_b(x)\}_{b=1}^{B} \quad \text{(分类)} \]

推导：为什么 Bagging 减少方差？

设每棵树的方差为 \(\sigma^2\)，树之间的相关系数为 \(\rho\)。则 \(B\) 棵树平均的方差为：

\[ \text{Var}\left(\frac{1}{B}\sum_{b=1}^{B} \hat{f}_b\right) = \frac{1}{B^2}\left(\sum_{b=1}^{B} \text{Var}(\hat{f}_b) + \sum_{b \neq c} \text{Cov}(\hat{f}_b, \hat{f}_c)\right) = \frac{\sigma^2}{B} + \frac{B-1}{B}\rho\sigma^2 \]

当 \(B \to \infty\)，方差趋近于 \(\rho\sigma^2\)。关键洞察：随机特征选择减少了树之间的相关性 \(\rho\)，从而进一步降低方差。这就是随机森林比单纯 Bagging 更好的原因。

8.2.2 袋外误差 (OOB Error)

每个 Bootstrap 样本大约包含 63.2% 的原始样本（因为 \(1 - (1-1/n)^n \approx 1 - 1/e \approx 0.632\)）。未被抽中的约 36.8% 样本称为 OOB（Out-Of-Bag）样本，可作为免费的验证集，无需额外划分训练/测试集。

竞赛技巧：随机森林的 OOB 误差通常能提供一个方便的内部泛化性能估计，在许多场景下接近交叉验证结果，但并非严格的「无偏估计」——在小样本、高维或弱效应等情况下，OOB 可能系统性偏离真实泛化误差（部分研究发现其可能高估真实错误率，即偏悲观）。在数模论文中报告 OOB 误差可以作为模型评估的参考，但重要结论仍建议配合独立测试集或交叉验证结果。此外，随机森林的特征重要性（基于平均不纯度减少或排列重要性）是分析变量影响力的重要工具。

8.2.3 随机森林 vs 单棵决策树

特性单棵决策树随机森林
可解释性极高（if-then 规则）低（黑箱）
方差高（对数据敏感）低（Bagging 降低方差）
偏差低略高（但通常可接受）
过拟合风险高（需要剪枝）低（树越多越稳定）
特征重要性基于分裂增益更稳定（平均多棵树）
计算复杂度\(O(pn\log n)\)\(O(B \cdot m_{\text{try}} \cdot n \log n)\)

其中 \(B\) 是树的数量，\(m_{\text{try}}\) 是每棵树每次分裂时随机选取的特征数。分类任务通常取 \(m_{\text{try}} \approx \sqrt{p}\)，回归任务通常取 \(m_{\text{try}} \approx p/3\)。实际复杂度还受树深、节点数、实现方式等影响，上表仅为典型渐近表达式。

8.2A 梯度提升：XGBoost 与 LightGBM

核心思想

梯度提升（Gradient Boosting）与随机森林不同——它不是并行训练多棵树然后平均，而是串行地训练树，每棵新树专门拟合前一轮的残差。最终预测是所有树的加权和。XGBoost 和 LightGBM 是梯度提升的工业级实现，在 Kaggle 和竞赛中统治表格数据。

梯度提升的数学原理

推导：梯度提升 = 函数空间的梯度下降

设损失函数为 \(L(y, F(x))\)，目标是找到 \(F(x)\) 最小化 \(\sum_i L(y_i, F(x_i))\)。

在第 \(m\) 轮，已经有 \(F_{m-1}(x)\)。计算负梯度（即残差的推广）：

\[ r_{im} = -\left[ \frac{\partial L(y_i, F(x_i))}{\partial F(x_i)} \right]_{F = F_{m-1}} \]

训练一棵回归树 \(h_m(x)\) 拟合 \(r_{im}\)。然后沿梯度方向更新：

\[ F_m(x) = F_{m-1}(x) + \nu \cdot h_m(x) \]

其中 \(\nu \in (0,1]\) 是学习率（shrinkage），控制每棵树的贡献。\(\nu\) 越小，需要的树越多，但泛化性能越好——这是 boosting 最重要的超参数。

XGBoost 的改进

XGBoost 在标准梯度提升上做了三个关键改进：

  1. 正则化目标函数：\(\text{Obj} = \sum_i L(y_i, \hat{y}_i) + \sum_m \Omega(h_m)\)，其中 \(\Omega(h) = \gamma T + \frac{1}{2}\lambda \|w\|^2\)，\(T\) 是叶子数，\(w\) 是叶子权重。这直接控制树的复杂度。
  2. 二阶泰勒展开：使用梯度和 Hessian（二阶导数）来更精确地近似损失函数，比只用一阶梯度的 GBDT 更准确。
  3. 列抽样 + 行抽样：类似随机森林的特征和样本随机化，进一步防止过拟合。
\[ \text{Gain} = \frac{1}{2}\left[ \frac{G_L^2}{H_L + \lambda} + \frac{G_R^2}{H_R + \lambda} - \frac{(G_L+G_R)^2}{H_L+H_R+\lambda} \right] - \gamma \]

其中 \(G = \sum g_i\)（梯度之和），\(H = \sum h_i\)（Hessian 之和）。分裂只有当 Gain > 0 时才执行——这天然实现了预剪枝。

LightGBM 的差异化优势

特性XGBoostLightGBM
树生长策略常见为 Level-wise（按层生长）Leaf-wise（按叶子生长）
连续特征处理直方图 / 近似分裂直方图分箱
类别特征当前版本支持原生类别特征原生类别特征
采样优化行列采样等Bagging / GOSS（样本选择）；EFB（稀疏特征捆绑）
速度取决于数据、参数和硬件，非固定倍数
竞赛技巧：XGBoost 和 LightGBM 是竞赛中必试的模型。调参顺序：① 先调 n_estimators（树的数量）和 learning_rate（通常 0.01-0.1）。② 再调 max_depth 和 min_child_weight。③ 最后调 subsample 和 colsample_bytree。④ 使用早停（early stopping）防止过拟合。LightGBM 在大数据上更快，XGBoost 在小数据上更稳定。

8.3 支持向量机 (SVM)

核心思想

SVM 寻找一个最大化间隔的超平面来分离两类样本。间隔（margin）定义为超平面到最近样本的距离。最大化间隔等价于最小化结构风险，这赋予了 SVM 出色的泛化能力。

8.3.1 硬间隔 SVM（线性可分情况）

设超平面为 \(w^T x + b = 0\)。对于线性可分的数据，要求所有样本满足 \(y_i(w^T x_i + b) \ge 1\)。几何间隔为 \(\frac{2}{\|w\|}\)。最大化间隔等价于：

\[ \min_{w,b} \frac{1}{2}\|w\|^2 \quad \text{s.t.} \quad y_i(w^T x_i + b) \ge 1, \quad \forall i \]

推导：SVM 的对偶问题

引入拉格朗日乘子 \(\alpha_i \ge 0\)：

\[ \mathcal{L}(w,b,\alpha) = \frac{1}{2}\|w\|^2 - \sum_{i=1}^{n} \alpha_i \left[ y_i(w^T x_i + b) - 1 \right] \]

对 \(w,b\) 求偏导并令其为 0，得 \(w = \sum \alpha_i y_i x_i\)，\(\sum \alpha_i y_i = 0\)。代入得对偶问题：

\[ \max_{\alpha} \sum_{i=1}^{n} \alpha_i - \frac{1}{2}\sum_{i=1}^{n}\sum_{j=1}^{n} \alpha_i \alpha_j y_i y_j (x_i^T x_j) \quad \text{s.t.} \quad \alpha_i \ge 0,\ \sum \alpha_i y_i = 0 \]

关键观察：① 对偶问题只涉及样本的内积 \(x_i^T x_j\)，这为核技巧铺平了道路。② 最优解中，只有支持向量（\(\alpha_i > 0\)）对决策边界有贡献——大多数 \(\alpha_i = 0\)，模型是稀疏的。

8.3.2 软间隔与核技巧

软间隔：当数据不完全线性可分时，引入松弛变量 \(\xi_i \ge 0\)：

\[ \min_{w,b,\xi} \frac{1}{2}\|w\|^2 + C \sum_{i=1}^{n} \xi_i \quad \text{s.t.} \quad y_i(w^T x_i + b) \ge 1 - \xi_i,\ \xi_i \ge 0 \]

其中 \(C\) 是正则化参数：\(C\) 大 → 更重视分类正确（可能过拟合），\(C\) 小 → 更重视间隔最大化（可能欠拟合）。

核技巧：将对偶问题中的内积 \(x_i^T x_j\) 替换为核函数 \(K(x_i, x_j)\)，隐式地在高维空间中进行线性分类：

核函数公式适用场景
线性核\(K(x_i,x_j) = x_i^T x_j\)高维稀疏数据（文本分类）
多项式核\(K(x_i,x_j) = (x_i^T x_j + c)^d\)已知多项式关系
RBF (高斯) 核\(K(x_i,x_j) = \exp(-\gamma\|x_i-x_j\|^2)\)最通用，竞赛首选
Sigmoid 核\(K(x_i,x_j) = \tanh(\kappa x_i^T x_j + c)\)等价于两层神经网络

竞赛工具箱：SVM 调参

核心参数：\(C\)（正则化强度）和 \(\gamma\)（RBF 核的带宽）。
调参策略：网格搜索 \(C \in \{10^{-3}, 10^{-2}, \ldots, 10^{3}\}\)，\(\gamma \in \{10^{-3}, \ldots, 10^{3}\}\)，配合交叉验证。
数据预处理：SVM 对特征尺度敏感——必须先标准化（Z-score 或 MinMax）。
竞赛模板：先用线性 SVM 做基线 → 再用 RBF SVM 调参 → 报告支持向量数量作为模型复杂度指标。

8.4 K 近邻 (KNN)

核心思想

KNN 可能是最简单的分类器：对新样本 \(x\)，找到训练集中距离它最近的 \(K\) 个样本，用多数投票决定类别。KNN 没有显式的训练过程——所有计算都在预测时进行（懒惰学习，lazy learning）。

8.4.1 距离度量

\[ \begin{aligned} \text{欧氏距离：} & d(x, x') = \sqrt{\sum_{j=1}^{p} (x_j - x_j')^2} \\ \text{曼哈顿距离：} & d(x, x') = \sum_{j=1}^{p} |x_j - x_j'| \\ \text{闵可夫斯基距离：} & d(x, x') = \left(\sum_{j=1}^{p} |x_j - x_j'|^q\right)^{1/q} \\ \text{余弦相似度：} & d(x, x') = 1 - \frac{x \cdot x'}{\|x\| \|x'\|} \end{aligned} \]
竞赛提示：高维空间中，所有点之间的距离趋于相等（维度灾难），KNN 失效。经验法则：当 \(p > 10\) 时，先做 PCA 降维或特征选择，再使用 KNN。此外，KNN 对特征尺度极度敏感——如果某个特征的取值范围是 [0,1000]，另一个是 [0,1]，距离几乎完全由第一个特征决定。必须标准化。

8.4.2 K 的选择

\(K\) 控制偏差-方差权衡：

  • \(K = 1\)：零偏差（训练误差为 0），但方差极大（决策边界非常曲折）
  • \(K\) 很大：决策边界平滑，但偏差增大（可能忽略局部模式）
  • 经验法则：\(K \approx \sqrt{n}\)，通过交叉验证精调

推导：KNN 的渐近性质

KNN 有两个重要的渐近结论，容易混淆：

① 固定 K，\(n \to \infty\)（Cover & Hart, 1967）：对于 1-NN，渐近错误率满足 \(R^* \le R_{1NN} \le 2R^*(1 - R^*)\)，即不超过贝叶斯错误率的两倍。对于一般固定 K，有更紧的界：

\[ R_{kNN} \le R^* \left(2 - \frac{K}{K-1}R^*\right) \]

② K → ∞，K/n → 0（Stone, 1977）：当 K 随样本量增长同时满足 \(K \to \infty\) 且 \(K/n \to 0\) 时，在适当条件下 KNN 是贝叶斯一致的——风险收敛到贝叶斯风险：

\[ R_{kNN} \to R^* \]

其中 \(R^*\) 是贝叶斯错误率（任何分类器能达到的最低错误率）。这两个结论给出了 KNN 强大的理论保证：足够大样本下，它是渐近一致的非参数分类器。

8.4B 朴素贝叶斯

核心思想

朴素贝叶斯基于贝叶斯定理，并做了一个「朴素」的假设——在给定类别的条件下，所有特征相互独立。尽管这个假设在现实中几乎不成立，但朴素贝叶斯在高维数据（如文本分类）中表现惊人地好，且训练速度极快。

贝叶斯定理与分类决策

\[ P(Y = k \mid X = x) = \frac{P(X = x \mid Y = k) \cdot P(Y = k)}{P(X = x)} \propto P(Y = k) \prod_{j=1}^{p} P(X_j = x_j \mid Y = k) \]

分类决策：选择后验概率最大的类别：\(\hat{y} = \arg\max_k P(Y=k) \prod_{j=1}^{p} P(X_j=x_j \mid Y=k)\)。

三种常见变体

变体\(P(X_j \mid Y=k)\) 的假设适用场景
Gaussian NB正态分布 \(N(\mu_{jk}, \sigma^2_{jk})\)连续特征，特征近似正态
Multinomial NB多项分布（词频）文本分类、离散计数
Bernoulli NB伯努利分布（出现/不出现）二元特征（如词袋模型）

推导：Gaussian NB 的参数估计

对于特征 \(j\) 和类别 \(k\)，用 MLE 估计均值和方差：

\[ \hat{\mu}_{jk} = \frac{1}{n_k} \sum_{i: y_i=k} x_{ij}, \quad \hat{\sigma}^2_{jk} = \frac{1}{n_k} \sum_{i: y_i=k} (x_{ij} - \hat{\mu}_{jk})^2 \]

预测时，计算每个类别的对数后验概率（取对数避免下溢）：

\[ \log P(Y=k \mid x) \propto \log P(Y=k) + \sum_{j=1}^{p} \left[ -\frac{1}{2}\log(2\pi\hat{\sigma}^2_{jk}) - \frac{(x_j - \hat{\mu}_{jk})^2}{2\hat{\sigma}^2_{jk}} \right] \]
竞赛技巧：朴素贝叶斯是绝佳的基线模型。训练只需计算均值和方差（O(np)），预测是 O(p)。在文本分类竞赛中，Multinomial NB + TF-IDF 是经典组合。此外，朴素贝叶斯的概率输出天然可解释——每个特征对最终分类的贡献可以精确计算。

8.5 分类方法选择决策树

数据高度非线性随机森林 / SVM (RBF)核方法或树模型能捕捉非线性模式
需要概率输出逻辑回归 / 随机森林逻辑回归直接输出概率，RF 可输出类别比例
需要报告特征重要性随机森林 / 逻辑回归RF 有内置的特征重要性度量

8.5A 类别不平衡处理

问题的本质

当正负样本比例严重失衡（如 1:99）时，分类器倾向于「全部预测为多数类」——准确率很高但完全无用。这个问题在竞赛中极为常见（欺诈检测、疾病诊断、罕见事件预测）。

SMOTE：合成少数类过采样

SMOTE (Synthetic Minority Oversampling TEchnique) 不是简单复制少数类样本，而是在特征空间中插值生成新样本：

SMOTE 算法

对每个少数类样本 \(x_i\)：
① 找到它的 \(k\) 个最近邻（KNN，在少数类内部）。
② 随机选择一个邻居 \(x_{nn}\)。
③ 在 \(x_i\) 和 \(x_{nn}\) 的连线上随机生成新样本：

\[ x_{\text{new}} = x_i + \lambda \cdot (x_{nn} - x_i), \quad \lambda \sim U(0,1) \]

这比简单复制好得多——新样本增加了少数类的「覆盖范围」，而不是在已有样本上重复。

其他处理方法

方法策略优点缺点
Class Weight给少数类更高的损失权重不改变数据分布需要调权重参数
Random Under-sampling随机丢弃多数类样本简单、数据量减少可能丢失重要信息
SMOTE合成少数类样本不丢失信息可能引入噪声
SMOTE + Tomek LinksSMOTE 后清理边界减少 SMOTE 的噪声计算量稍大
Threshold Moving调整分类阈值最简单、无额外计算不改变模型本身
竞赛常见错误：① 不标准化就使用 SVM/KNN（距离和核函数对尺度敏感）。② 用随机森林却无法解释结果——评委需要知道哪些特征最重要。③ 在极度不平衡的数据上只报告准确率——需要看精确率、召回率、F1（见 §X）。④ 没有报告交叉验证结果——单次训练/测试划分的随机性太大。⑤ 在测试集上做 SMOTE——SMOTE 只能用于训练集，测试集必须保持原始分布。

8.5B AdaBoost

Boosting 的始祖

AdaBoost (Adaptive Boosting) 是第一个实用的 boosting 算法（Freund & Schapire, 1995）。与梯度提升不同，AdaBoost 通过迭代调整样本权重来训练弱分类器序列——每个新分类器专注于前一个分类器分错的样本。

AdaBoost 算法

Step 1：初始化所有样本权重 \(w_i = 1/n\)。
Step 2：对于 m = 1 到 M：

  1. 用当前权重 \(w_i\) 训练弱分类器 \(G_m(x)\)
  2. 计算加权误差 \(err_m = \sum w_i \cdot \mathbf{1}\{G_m(x_i) \neq y_i\} / \sum w_i\)
  3. 计算分类器权重 \(\alpha_m = \ln\left(\frac{1 - err_m}{err_m}\right)\)
  4. 更新样本权重：\(w_i \leftarrow w_i \cdot \exp(\alpha_m \cdot \mathbf{1}\{G_m(x_i) \neq y_i\})\)

Step 3：最终分类器：\(G(x) = \text{sign}\left(\sum_{m=1}^{M} \alpha_m G_m(x)\right)\)

\[ \alpha_m = \frac{1}{2}\ln\left(\frac{1 - err_m}{err_m}\right) \quad \text{——误差越小的分类器，权重越大} \]
竞赛对比：AdaBoost 对异常值敏感（因为误分类样本的权重指数增长），适合数据干净、噪声低的场景。XGBoost/LightGBM 对噪声更鲁棒，是竞赛中的默认选择。但 AdaBoost 概念简单、理论优美，在论文中作为对比方法很有价值。

8.5C 超参数调优

模型性能的「隐藏开关」

超参数（Hyperparameters）是在训练前设定的参数——如 KNN 的 K、随机森林的树数、SVM 的 C 和 γ、XGBoost 的学习率。选择合适的超参数可以显著提升模型性能，竞赛中必须做超参数调优。

Grid Search (网格搜索)

预先定义每个超参数的候选值集合，穷举所有组合，通过交叉验证选择最优组合。例如：KNN 的 K ∈ {1,3,5,7,9,11}，SVM 的 C ∈ {0.1,1,10,100}，γ ∈ {0.01,0.1,1} → 共 6×4×3=72 种组合。

Random Search (随机搜索)

从超参数空间中随机采样一定数量的组合。研究表明，在相同计算预算下，随机搜索通常优于网格搜索——因为不是所有超参数都同等重要，随机搜索能探索更多重要超参数的值。

Bayesian Optimization (贝叶斯优化)

用高斯过程（或其他代理模型）建模超参数与 CV 性能之间的关系，每次选择「最可能提升性能」的超参数组合来评估。贝叶斯优化比随机搜索更高效（用更少的评估次数找到更好的超参数），但实现更复杂。

建模工具箱：超参数调优指南

模型关键超参数调优范围

竞赛陷阱：① 在测试集上调超参数——这是数据泄漏，会导致性能虚高。正确的做法：训练集 → 交叉验证调参 → 验证集评估 → 测试集最终报告。② 超参数调优不报告搜索范围和 CV 折数——论文中必须写清楚。③ 调参范围太窄——最优值可能在你设定的范围之外。

8.5D CatBoost

专为分类特征设计的 GBDT

CatBoost (Categorical Boosting, Yandex 2017) 与 XGBoost/LightGBM 同属 GBDT 家族，但其核心创新是对分类特征的自动处理——不需要手动做 one-hot 编码或标签编码。

CatBoost 的三个关键创新：

  1. 有序目标统计 (Ordered Target Statistics)：将分类特征的值替换为该类别的目标均值——但使用排序后的历史数据计算，避免数据泄漏（传统目标编码会导致过拟合）。
  2. 有序提升 (Ordered Boosting)：每棵树只用前 i-1 个样本训练，预测第 i 个样本——避免梯度估计的偏差。
  3. 对称树 (Symmetric Trees)：与 XGBoost 的非对称树不同，CatBoost 使用对称（平衡）树——所有叶子在同一深度分裂，预测速度更快。
竞赛选择：当数据中分类特征很多（如城市、职业、产品类别等）且不想手动编码时，CatBoost 值得优先尝试（原生类别特征支持最完善）。当特征主要是数值型时，LightGBM 通常更快，XGBoost 更稳定。三者性能差异通常很小——竞赛中都值得尝试，选交叉验证最好的即可。

8.6 竞赛实战流程

分类问题竞赛流程

Step 1：数据预处理——标准化（SVM/KNN 必需）、处理缺失值、编码分类变量。
Step 2：探索性分析——各类别在特征空间中的分布（PCA 降维可视化，见 §X）。
Step 3：基线模型——逻辑回归或 KNN（简单快速，建立性能基准）。
Step 4：进阶模型——随机森林或 SVM（RBF 核），调参 + 交叉验证。
Step 5：模型比较——报告准确率、精确率、召回率、F1、AUC（见 §X）。
Step 6：特征重要性分析——哪些特征对分类贡献最大？用随机森林的特征重要性。
Step 7：结果解释——不只报告数字，要解释为什么选择这个模型（结合数据特征，而非仅凭指标排序）。

🔬 案例：玻璃类型的分类

以14种化学成分为特征，分类目标：高钾玻璃 vs 铅钡玻璃。决策树用PbO>5%为关键分裂点（直观可解释）。随机森林特征重要性：PbO>BaO>K₂O>SiO₂。测试集准确率>95%。这是典型的高维小样本分类问题。
