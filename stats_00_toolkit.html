数模实战工具箱

Pandas 数据处理速查、NumPy 数值计算、Matplotlib 六图绘制、模型验证完整流程（交叉验证 / 残差分析 / 敏感性 / 鲁棒性）、论文表达六问框架。建模之前建议熟练的基础技能。

📋 第一层能力清单

以下技能在竞赛中建议达到熟练级别——频繁查文档会拖慢节奏。每一条都是真实赛题中反复出现的基础操作。

0.1 Pandas 数据处理速查

Pandas 是数模竞赛最常用的工具之一。赛题数据几乎都是 Excel/CSV 格式，Pandas 是 Python 生态的核心入口（R、MATLAB 等也是可行选择）。以下 7 个操作覆盖了 80% 的竞赛数据处理需求。

0.1.1 读取数据

# 读取 Excel 和 CSV —— 竞赛数据最常见的两种格式 import pandas as pd # Excel：注意 sheet_name 参数 —— 赛题数据经常分多个 sheet df = pd.read_excel('data.xlsx', sheet_name='Sheet1') df = pd.read_excel('data.xlsx', sheet_name=None) # 读所有 sheet → dict # CSV：注意 encoding —— 中文数据经常是 GBK df = pd.read_csv('data.csv', encoding='utf-8') df = pd.read_csv('data.csv', encoding='gbk') # 中文 CSV 常用 # 读完后第一件事： df.head() # 前 5 行 df.info() # 列名 + 类型 + 缺失值 df.describe() # 数值列的五数概括
竞赛陷阱：中文 CSV 文件默认编码通常是 GBK，不是 UTF-8。如果 read_csv 报错 UnicodeDecodeError，先试 encoding='gbk'，再试 'gb2312'，最后试 'latin-1'（后者不会报错但中文会乱码，仅用于确认编码问题）。

0.1.2 数据合并

# merge —— 类似 SQL JOIN，按关键列连接两张表 df = pd.merge(left, right, on='ID', how='left') # how: 'left' | 'right' | 'inner' | 'outer' # 多列连接 df = pd.merge(a, b, on=['year', 'city']) # concat —— 纵向堆叠或横向拼接 df = pd.concat([df1, df2], axis=0) # 纵向：追加行 df = pd.concat([df1, df2], axis=1) # 横向：追加列
merge vs concat：merge 按「键」连接（需要共同的列），concat 按「位置」堆叠。赛题中多表关联用 merge，同一张表的分段数据用 concat。

0.1.3 分组聚合

# groupby —— 分组后聚合，数模中最常用的操作之一 df.groupby('city')['sales'].sum() # 每个城市的销售额总和 df.groupby('city')['sales'].mean() # 每个城市的平均销售额 # agg —— 同时计算多个统计量 df.groupby('city').agg({ 'sales': ['sum', 'mean', 'count'], 'price': ['min', 'max'] }) # transform —— 保持原形状的分组计算（用于特征工程） df['sales_z'] = df.groupby('city')['sales'].transform( lambda x: (x - x.mean()) / x.std() )

0.1.4 数据透视

# pivot_table —— 从「长表」变「宽表」 pt = pd.pivot_table(df, values='sales', # 要聚合的值 index='city', # 行索引 columns='month', # 列索引 aggfunc='sum', # 聚合函数 fill_value=0 # 缺失值填充 ) # crosstab —— 频数交叉表（分类变量专用） ct = pd.crosstab(df['gender'], df['outcome'], normalize='index') # 行百分比
pivot_table vs crosstab：pivot_table 用于数值聚合（求和、平均等），crosstab 用于计数/频率。两者底层都是 groupby，但语法更简洁。

0.1.5 数据清洗

# 缺失值处理 —— 竞赛数据几乎都有缺失 df.isnull().sum() # 每列缺失值计数 df = df.fillna(0) # 填充为 0 df = df.fillna(df.mean()) # 填充为列均值 df = df.ffill() # 前向填充（时间序列常用；pandas 2.1+） df = df.bfill() # 后向填充 # 插值（比简单填充更合理） df['col'] = df['col'].interpolate() # 线性插值 # 去重 df = df.drop_duplicates() # 完全重复的行 df = df.drop_duplicates(subset=['ID']) # 按某列去重，保留第一个 df = df.drop_duplicates(subset=['ID'], keep='last') # 异常值过滤（3σ 原则） mean, std = df['col'].mean(), df['col'].std() df = df[(df['col'] > mean - 3*std) & (df['col'] < mean + 3*std)]
竞赛陷阱：缺失值处理没有固定优先级，应首先根据变量含义和缺失机制判断。未检出值（如仪器检测下限以下）填 0 或 LOD/√2 可能是正确操作；时间序列可用插值；随机缺失可用中位数/均值填充或多重插补；信息完全缺失只能删除。无论采用何种方法，论文中应说明理由并通过敏感性分析比较不同处理方式对结论的影响。

# 统计函数 x = np.array([1, 2, 3, 4, 5]) y = np.array([2, 4, 5, 4, 5]) np.mean(x); np.median(x); np.std(x); np.var(x) np.min(x); np.max(x); np.percentile(x, 25) np.corrcoef(x, y) # 相关系数矩阵 np.histogram(x, bins=20) # 直方图计数 # 随机抽样（蒙特卡洛、bootstrap 的基础） np.random.seed(42) # 固定随机种子（论文可复现） np.random.rand(100) # [0,1) 均匀分布 np.random.randn(100) # 标准正态分布 np.random.randint(0, 10, 50) # [0,10) 随机整数 arr = np.array([1, 2, 3, 4, 5]) np.random.choice(arr, size=3, replace=True) # 有放回抽样 np.random.permutation(arr) # 随机排列（交叉验证打乱数据）

数模论文绘图最佳实践

  1. 每张图应有标题和轴标签——缺少标签的图直接扣分
  2. 字体不能太小——论文中的图经常被缩小，确保 8pt 以上

  4. 分辨率 300 dpi——plt.savefig('fig.png', dpi=300, bbox_inches='tight')
  5. 图不要截图——用 savefig 导出，不要用截图工具截 Matplotlib 窗口
  6. 中文问题——SimHei 黑体是最稳妥的中文字体选择

0.4 模型验证完整流程

这是初学者最容易忽略的部分。建立一个模型只完成了 30% 的工作——验证模型的可靠性、评估其局限、确保其可复现，才是剩下的 70%。

0.4.1 训练集与测试集

from sklearn.model_selection import train_test_split X_train, X_test, y_train, y_test = train_test_split( X, y, test_size=0.3, random_state=42, stratify=y # 分类问题必加：保持训练/测试集的类别比例一致 ) # 关键参数： # test_size: 测试集比例（常用 0.2-0.3） # random_state: 固定随机种子，保证可复现 # stratify=y: 分类问题保持类别比例（不平衡数据集必须使用）
竞赛陷阱：数据量小时（n < 200），单次划分的随机性很大——训练集和测试集的「运气」会影响结论。此时应该用交叉验证代替单次划分，或至少报告多次随机划分的平均结果。

0.4.2 误差指标

指标公式适用场景特点
MSE\(\frac{1}{n}\sum (y_i-\hat{y}_i)^2\)回归（通用）对大误差敏感（平方惩罚）
RMSE\(\sqrt{\text{MSE}}\)回归（通用）与原数据同单位，更直观
MAE\(\frac{1}{n}\sum |y_i-\hat{y}_i|\)回归（有离群值）对异常值不敏感
MAPE\(\frac{100\%}{n}\sum \left|\frac{y_i-\hat{y}_i}{y_i}\right|\)回归（可解释性）百分比误差；\(y_i\) 接近 0 时失效
R²\(1-\frac{\sum (y_i-\hat{y}_i)^2}{\sum (y_i-\bar{y})^2}\)回归（拟合优度）越接近 1 越好；可能为负
Accuracy\(\frac{\text{TP+TN}}{\text{Total}}\)分类（均衡）类别不均衡时不可靠
F1\(\frac{2\cdot\text{Precision}\cdot\text{Recall}}{\text{Precision+Recall}}\)分类（不均衡）精确率与召回率的调和平均
AUCROC 曲线下面积二分类（排序质量）0.5=随机, 1.0=完美
竞赛建议：不要只报告一个指标。回归通常报告 RMSE + MAE + R²（三者互补：RMSE 惩罚大误差，MAE 更稳健，R² 反映解释比例）；分类通常报告 Accuracy + F1 + AUC（但对多分类问题 AUC 需谨慎使用，类别极不均衡时 PR-AUC 可能比 ROC-AUC 更有意义，有明确误判成本时应报告成本敏感指标）。单一指标容易造成误导——比如在极度不均衡的数据上，Accuracy 可以很高但模型毫无用处。

0.4.3 交叉验证

from sklearn.model_selection import cross_val_score, KFold, StratifiedKFold # K 折交叉验证 —— 标准做法 scores = cross_val_score(model, X, y, cv=5, scoring='neg_mean_squared_error') print(f'CV MSE: {-scores.mean():.4f} ± {scores.std():.4f}') # KFold —— 自定义折数 kf = KFold(n_splits=5, shuffle=True, random_state=42) for train_idx, test_idx in kf.split(X): X_train, X_test = X[train_idx], X[test_idx] # ... 训练和评估 ... # StratifiedKFold —— 分类问题保持每折的类别比例 skf = StratifiedKFold(n_splits=5, shuffle=True, random_state=42)

为什么用交叉验证而不是单次划分？

单次 train/test split 的评估结果依赖于随机种子的「运气」。5 折交叉验证给出 均值 ± 标准差，评委可以看到：① 模型平均表现如何；② 表现是否稳定（标准差小 = 稳定）。如果标准差很大，说明模型对数据划分敏感——这是需要讨论的问题，不是隐藏的 bug。

# 残差 = 真实值 - 预测值 residuals = y_test - y_pred # 残差诊断三件套： # ① 残差 vs 拟合值图 —— 检查异方差和非线性 plt.scatter(y_pred, residuals, alpha=0.6) plt.axhline(y=0, color='r', linestyle='--') # ② Q-Q 图 —— 检查残差正态性 from scipy import stats stats.probplot(residuals, dist='norm', plot=plt) # ③ 残差直方图 —— 检查分布形态 plt.hist(residuals, bins=30, edgecolor='white')

0.4.5 敏感性分析

敏感性分析的核心思想

模型中的参数（权重、阈值、假设值）往往不是精确已知的。敏感性分析回答：如果参数变化 ±10%，结论会变吗？如果微调参数就导致结论翻转，说明模型不可靠——评委一定会指出这一点。

# 敏感性分析的标准做法：参数扫描 def sensitivity_analysis(model_fn, param_name, base_value, range_pct=0.2, steps=21): """对 model_fn 的某个参数做 ±range_pct 的扫描""" results = [] for pct in np.linspace(-range_pct, range_pct, steps): param_value = base_value * (1 + pct) result = model_fn(**{param_name: param_value}) results.append((pct, result)) return results # 论文中建议报告： # ① 参数变化范围（±10%、±20% 等） # ② 输出指标的变化幅度 # ③ 是否存在「临界点」（参数过某个值后结论翻转）

0.4.6 鲁棒性分析

敏感性分析 vs 鲁棒性分析

敏感性分析问的是「参数变了，结果变多少？」——它关注连续、小幅的参数扰动。
鲁棒性分析问的是「如果假设被打破，模型还成立吗？」——它关注结构性变化：数据分布变了、删掉一些变量、换一种建模方法、引入噪声。两者互补，论文中应同时出现。

# 鲁棒性分析工具箱 # ① Bootstrap 鲁棒性 —— 重抽样看系数稳定性 def bootstrap_coef(X, y, model_class, n_boot=1000): coefs = [] for _ in range(n_boot): idx = np.random.choice(len(X), len(X), replace=True) model = model_class().fit(X[idx], y[idx]) coefs.append(model.coef_) return np.mean(coefs, axis=0), np.std(coefs, axis=0) # ② 加噪声鲁棒性 —— 给输入加随机扰动 X_noisy = X + np.random.normal(0, noise_std, X.shape) score_noisy = model.score(X_noisy, y) # ③ 变量删除鲁棒性 —— 逐个删除变量看影响 for col in X.columns: X_drop = X.drop(columns=[col]) score = cross_val_score(model, X_drop, y, cv=5).mean() print(f'Drop {col}: score={score:.4f}') # ④ 方法替换鲁棒性 —— 换一种模型看结论是否一致 from sklearn.linear_model import LinearRegression, Ridge, Lasso for name, model in [('OLS', LinearRegression()), ('Ridge', Ridge()), ('Lasso', Lasso())]: score = cross_val_score(model, X, y, cv=5).mean() print(f'{name}: {score:.4f}')

鲁棒性分析写作模板

「为验证模型的鲁棒性，我们从以下三个维度进行了检验：(1) 数据鲁棒性——通过 Bootstrap 重抽样，模型系数的变异系数均小于 10%，表明估计稳定；(2) 噪声鲁棒性——在输入数据中加入 5% 的高斯噪声后，预测误差仅上升 3.2%，表明模型对测量误差不敏感；(3) 方法鲁棒性——改用 Ridge 回归和 Lasso 回归后，关键变量的符号和显著性均保持一致，表明结论不依赖于特定模型形式。」

0.5 论文表达六问框架

每个模型在论文中出现时，建议能回答以下六个问题。这是评委判断「这个模型是认真选的还是随便套的」的核心依据。

QUESTION 01
为什么使用它？
不能只说「因为它是经典方法」。要说明：你的数据特征、问题结构、假设条件，与这个模型的适用场景匹配在哪里。例如：「由于因变量是二分类且自变量间存在多重共线性，选择带 L2 正则化的 Logistic 回归而非普通线性回归。」
QUESTION 02
输入和输出是什么？
用一张表或一段话说清楚：输入是什么（几个变量、什么类型、什么范围）、输出是什么（数值、分类、概率）。评委需要快速理解模型在做什么。
QUESTION 03
有哪些假设？
每个模型都有假设。列出 2-4 条，并说明你的数据是否满足。不满足的部分要说明：「虽然假设 X 不完全满足，但通过 Y 方法做了缓解。」假装没看到假设是论文中最常见的扣分点。
QUESTION 04
公式中的变量代表什么？
公式出现后，立即用一个表格解释每个符号的含义和单位。不要假设评委能猜到。尤其注意：数学符号和代码变量名要一一对应。
QUESTION 05
结果是否可信？
用交叉验证、残差分析、敏感性分析、鲁棒性分析来支撑。不能只说「R²=0.92」，而要展示：「5 折交叉验证 R²=0.92±0.03；参数 ±20% 扰动后结论不变。」
QUESTION 06
模型有什么缺点？怎么说？
主动承认缺点比被动被发现更好。用「局限性」而非「缺陷」这个词。例如：「本模型的局限性在于：(1) 假设线性关系，对非线性模式捕捉能力有限；(2) 对异常值敏感，已在预处理中做了 Winsorize 处理。未来可考虑引入非线性项或使用集成方法。」

「局限性」写作建议

每个局限性 = 一个具体问题 + 一个已做的缓解措施 + 一个未来的改进方向。不要只写「模型可能不够好」——这等于什么都没说。正确的写法是：「模型假设残差独立，但时间序列数据可能存在自相关（问题）。我们通过 Durbin-Watson 检验确认了残差独立性（缓解）。若 DW 统计量不理想，可采用 GLS 或引入滞后项（改进）。」
